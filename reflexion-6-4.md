git merge para fusionar dos ramas





git rebase reescribe el historial, moviendo los commits de una rama para que se apliquen encima de otra

git rebase cuando estoy trabajando en una rama local de funcionalidad y quiero mantener un historial limpio y ordenado antes de integrarla en main.


git merge cuando trabajo en equipo y las ramas ya han sido compartidas

git rebase en ramas que ya han sido subidas al repositorio remoto porque reescribe el historial de commits.
